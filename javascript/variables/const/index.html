const — ligação fixa, escopo firme

1. Declaração obrigatória com valor

const nome = "Sanciweferson"
Resultado →
const nasce completo. Não existe undefined inicial.

2. Reatribuição bloqueada

nome = "Outro"
Resultado →
A ligação é fixa. Reatribuir gera erro imediatamente.

3. Escopo de bloco

if (true) {
  const idade = 20
}
console.log(idade)
Resultado →
const respeita escopo de bloco. Nada vaza para fora.

4. Hoisting e TDZ

console.log(x)
const x = 10
Resultado →
Existe hoisting, mas a variável fica na Temporal Dead Zone.

5. const com objeto (mutação permitida)

const pessoa = { nome: "San" }
pessoa.nome = "Sanciweferson"
Resultado →
O objeto muda. A referência não.

const não congela dados.

6. Tentando trocar o objeto

pessoa = { nome: "João" }
Resultado →
Trocar a referência quebra a regra do const.

7. const no loop (closures corretas)

const funcoes = []

for (const i of [0,1,2,3]) {
  funcoes.push(() => i)
}
